9 ## 1. Introduction
10 
11 The unitree_legged_sdk is mainly used for communication between PC and Controller board.
12 It also can be used in other PCs with UDP.
13 
14 - **notics**: support robot: Go1, not support robot: Laikago, Aliengo, A1. (Check release [v3.2](https://github.com/unitreerobotics/unitree_legged_sdk/releases/tag/v3.2) for support)
15 
16 ### 1.1 Usage
17 
18 There are four steps that use this software development kit (SDK) to control robot.
19 
20 - Communication
21 
22 - Dependencies
23 - Build
24 - Run

26 #### 1.1.1 Communication
27 
28 Establish communication between PC and Controller board. If you‘re using the PC(Raspberry in the red box below) on Go1, you can skip this section.
29 
30 ![](/home/david/Documents/unitree/github/unitree_legged_sdk/document/image/Go1CommFram.svg)
31 
32 - Use own PC(with Ubuntu system) USB port to connect Go1's Ethernet port.
33 
34  <img src="document/image/USB2Ethernet.svg"/>
35 
36  Then open terminal and execute following command:
37 
38  ```bash
39  ifconfig # Run this command after you plug in the USB porthub, you can find an extra device ID. For example, enpxxx
40  sudo ifconfig enpxxx down # enpxxx is your PC usb port
41  sudo ifconfig enpxxx 192.168.123.162/24
42  sudo ifconfig enpxxx up
43  ping 192.168.123.161 # if you can recive message like "64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=xxx ms", you are connected
44  ```
45 
46 - Use own PC(with Ubuntu system) Ethernet port to connect Go1's Ethernet port.
47 
48  <img src="document/image/Ethernet2Ethernet.svg"/>
49  Then open terminal and execute following command:
50 
51  ```bash
52  sudo ifconfig eth0 down # eth0 is own PC Ethernet port
53  sudo ifconfig eth0 192.168.123.162/24
54  sudo ifconfig eth0 up
55  ping 192.168.123.161 # if you can recive message like "64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=xxx ms", you are connected
56  ```

58 #### 1.1.2 Dependencies
59 
60 If you‘re using the PC on Go1, you can skip this section.
61 
62 * [Boost](http://www.boost.org) (version 1.5.4 or higher)
63 * [CMake](http://www.cmake.org) (version 2.8.3 or higher)
64 * [LCM](https://lcm-proj.github.io) (version 1.4.0 or higher)
65 
66 ```bash
67 # Download LCM package, then:
68 cd lcm-x.x.x
69 mkdir build
70 cd build
71 cmake ../
72 make
73 sudo make install
74 ```

77 
78 Open the terminal in the unitree_legged_sdk folder and execute following command.
79 
80 ```bash
81 mkdir build
82 cd build
83 cmake ../
84 make
85 ```

87 #### 1.1.4 Run
88 
89  Open the terminal in the "build/bin" folder to run the binary file.
90 
91 - high-level example can be run directly, for example:
92 
93 ```bash
94 # Run examples with 'sudo' for memory locking.
95 sudo ./example_walk
96 ```
97 
98 - Before running the low-level examples, please switch Go1's control mode to basic mode using wireless handle.
99 
100 ```bash
101 # WARNING: Make sure the robot is hung up.
102 sudo ./example_postion
103 sudo ./example_torque
104 sudo ./example_velocity
105 ```

122 The files in *"include/unitree_legged_sdk/"* defines constant, variable, class, API functions, etc.
123 
124 | include/unitree_legged_sdk/ | description |
125 | ---------------------------------------- | ------------------------------------------------------------ |
126 | a1_const.h aliengo_const.h go1_const.h | Defines the limits of the robot's 12 joint motors |
127 | quadruped.h | Defines robot type, control level type, leg nunber and joint number |
128 | unitree_joystick.h | Defines key and joystick variable data type |
129 | comm.h | defines common class, such as LowState, LowCmd, HighState and HighCmd class, to store user command or robot sates data |
130 | safety.h | Defines Safety class, keep the robot in a safe environment |
131 | udp.h | Defines UDP class, user can communication between PC and Controller board through UDP object, it also can be used in other PCs |
132 | lcm.h | Defines LCM class, user can send command and receive robot states data on one PC through LCM object |
133 | lcm_server.h | Defines Lcm_Server_Low class and Lcm_Server_High class, except communication through UDP object, user can also communication through LCM object |
134 | loop.h | Defines LoopFunc class, users can generate different threads for different objects through LoopFunc object |
135 | unitree_legged_sdk.h | Included all of the above header files |
136

138

141 | example_position.cpp | Example of low-level position control of robot, the calf of right front leg will flapping | Basic mode |
142 | example_torque.cpp | Example of low-level torque control of robot, the thigh joint of right front leg will move to zero position and be in impedance control state | Basic mode |
143 | example_velocity.cpp | Example of low-level velocity control of robot, the calf of right front leg will flapping | Basic mode |
144 | example_walk.cpp | Example of high-level control of robot, the robot will do a series of movements | Sport mode |
145 | example_wirelessHandle.cpp | Example of wireless handle communication, joystick data will be printed on the screen When key A is pressed | Basic mode |
146 | lcm_server.cpp | Example of LCM communication server, contains low-level and high-level communication | |
147 | multi_pc_udp_recv.cpp | Example of UDP communication server, running client(multi_pc_udp_send.cpp) on the another PC | |
148 | multi_pc_udp_send.cpp | Example of UDP communication client, running server(multi_pc_udp_send.cpp) on another PC. | |
